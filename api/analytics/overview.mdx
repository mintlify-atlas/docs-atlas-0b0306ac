---

## Overview

This endpoint provides comprehensive analytics data for a form. The response includes basic metrics for STANDARD plan users and extended analytics including browser statistics and location data for PRO plan users.

<Info>
  Analytics are only available with the STANDARD or PRO plan. FREE plan users will receive a 403 error.
</Info>

## Authentication

This endpoint requires a valid API key. The API key can be provided in two ways:

1. **Header (Recommended)**: `X-API-Key: your_api_key`
2. **Query Parameter**: `?apiKey=your_api_key`

## Path Parameters

<ParamField path="formId" type="string" required>
  The unique identifier of the form to retrieve analytics for.
</ParamField>

## Query Parameters

<ParamField query="apiKey" type="string">
  API key for authentication. Only required if not provided in the `X-API-Key` header.
</ParamField>

<ParamField query="timeRange" type="string" default="week">
  Time range for the time series data.
  
  <Expandable title="Available values">
    - `day` - Hourly data for the last 24 hours
    - `week` - Daily data for the last 7 days
    - `month` - Daily data for the last 30 days
  </Expandable>
</ParamField>

## Response

<ResponseField name="totalSubmissions" type="number">
  Total number of submissions received for this form.
</ResponseField>

<ResponseField name="dailySubmissionRate" type="number">
  Average number of submissions per day since the form was created. Rounded to 2 decimal places.
</ResponseField>

<ResponseField name="last24HoursSubmissions" type="number">
  Number of submissions received in the last 24 hours.
</ResponseField>

<ResponseField name="timeSeriesData" type="array">
  Array of time series data points showing submission counts over the specified time range.
  
  <Expandable title="properties">
    <ResponseField name="time" type="string">
      Time label for the data point. Format depends on timeRange:
      - `day`: "0:00", "1:00", etc. (24 hour format)
      - `week`: "Mon", "Tue", etc. (day names)
      - `month`: "3/1", "3/2", etc. (month/day format)
    </ResponseField>
    
    <ResponseField name="submissions" type="number">
      Number of submissions during this time period.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="timeRange" type="string">
  The time range used for the analytics data (echoes the query parameter).
</ResponseField>

<ResponseField name="plan" type="string">
  The user's current plan: `STANDARD` or `PRO`.
</ResponseField>

### PRO Plan Only Fields

The following fields are only included for PRO plan users:

<ResponseField name="weekOverWeekGrowth" type="number">
  Week-over-week growth rate calculated as the percentage change between the last 7 days and the previous 7 days. Rounded to 2 decimal places.
</ResponseField>

<ResponseField name="peakSubmissionHour" type="number">
  Hour of the day (0-23) when submissions are most frequently received.
</ResponseField>

<ResponseField name="browserStats" type="array">
  Statistics on which browsers were used to submit the form.
  
  <Expandable title="properties">
    <ResponseField name="name" type="string">
      Browser name (e.g., "Chrome", "Firefox", "Safari", "Edge").
    </ResponseField>
    
    <ResponseField name="count" type="number">
      Number of submissions from this browser.
    </ResponseField>
    
    <ResponseField name="percentage" type="number">
      Percentage of total submissions from this browser (0-1 range).
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="locationStats" type="array">
  Statistics on submission locations by country.
  
  <Expandable title="properties">
    <ResponseField name="name" type="string">
      Country name (e.g., "United States", "United Kingdom").
    </ResponseField>
    
    <ResponseField name="count" type="number">
      Number of submissions from this country.
    </ResponseField>
    
    <ResponseField name="percentage" type="number">
      Percentage of total submissions from this country (0-1 range).
    </ResponseField>
  </Expandable>
</ResponseField>

## Rate Limiting

This endpoint is rate-limited. When rate limits are exceeded, you'll receive a 429 status code with the following headers:

- `X-RateLimit-Limit`: Maximum number of requests allowed
- `X-RateLimit-Remaining`: Number of requests remaining
- `X-RateLimit-Reset`: Unix timestamp when the rate limit resets

## Examples

<CodeGroup>

```bash cURL
curl -X GET "https://api.mantlz.com/api/v1/forms/form_123/analytics?timeRange=week" \
  -H "X-API-Key: your_api_key"
```

```javascript JavaScript
const response = await fetch(
  'https://api.mantlz.com/api/v1/forms/form_123/analytics?timeRange=week',
  {
    method: 'GET',
    headers: {
      'X-API-Key': 'your_api_key'
    }
  }
);

const analytics = await response.json();
console.log(analytics);
```

```python Python
import requests

response = requests.get(
    'https://api.mantlz.com/api/v1/forms/form_123/analytics',
    params={'timeRange': 'week'},
    headers={'X-API-Key': 'your_api_key'}
)

analytics = response.json()
print(analytics)
```

</CodeGroup>

## Error Responses

<ResponseField name="error" type="string">
  Error message describing what went wrong.
</ResponseField>

### Common Errors

| Status Code | Error Message                                              | Description                                          |
| ----------- | ---------------------------------------------------------- | ---------------------------------------------------- |
| 400         | Invalid timeRange parameter                                | timeRange must be "day", "week", or "month"          |
| 401         | Invalid or inactive API key                                | The provided API key is invalid or has been revoked  |
| 403         | Analytics are only available with the STANDARD or PRO plan | User is on the FREE plan which doesn't include analytics |
| 404         | Form not found                                             | The form doesn't exist or doesn't belong to the user |
| 429         | Rate limit exceeded                                        | Too many requests, wait for rate limit to reset      |
| 500         | Internal server error                                      | Server encountered an error processing the request   |
